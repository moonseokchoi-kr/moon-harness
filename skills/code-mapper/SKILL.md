# code-mapper — ephemeral 구조적 컨텍스트 제공기

작업 직전 대상 심볼의 **실제 호출관계**를 고정 커버리지 체크리스트로 surface하고, 컨텍스트로 소비된 뒤 버려지는 도구다 (Aider repo map 모델). grep(이름 매칭)만으로 코드를 파악하면 "이름이 맞으니 내가 원하는 동작이겠지"라는 **가짜 확신**이 생긴다. code-mapper는 그 확신을 근거 있는 이해로 바꾼다.

## 정체성 (변하면 안 되는 축)

- **ephemeral**: 산출물은 매 호출마다 재생성되어 컨텍스트로만 소비되고 버려진다. 파일로 저장하지 않는다 (`docs/`나 task 문서에 코드맵을 기록하지 않는다).
- **게이트가 아니라 컨텍스트**: 산출물 준수를 강제·검증·차단하지 않는다. 더 나은 컨텍스트를 적시에 줄 뿐이다.
- **금지**: 저장(영속) / 계약 / 검증 / 종속 / 하드게이트 전부 비목표. (상세는 맨 아래 "비목표" 섹션)
- **하네스 범용**: 레포 특화 명령·경로·컨벤션을 이 절차에 하드코딩하지 않는다.

## 호출

```
/code-mapper <심볼-또는-파일>
```

사람이 직접 호출하거나, 버그 수정 근본원인 분석·SDD 구현 등에서 낯선 심볼을 만지기 전 컨텍스트 확보용으로 호출한다. 이 SKILL.md가 절차의 **단일 진실 공급원(SSOT)** 이다 — 다른 파일은 절차 본문을 복제하지 않고 포인터만 가진다.

## 절차

단일 동기 절차. 동시성·백그라운드 작업 없음. 핵심 흐름: **프로브 → 분기 → surface → 폐기**.

### Step 1 — codegraph 가용성 프로브 (`codegraph_status`)

- **`codegraph_status` MCP 도구를 호출**해 그 응답으로만 가용성을 판정한다.
- **filesystem `.codegraph/` 디렉터리 존재 여부 체크 금지.** codegraph가 순수 MCP 형태로 전환 중이라 로컬 인덱스 폴더 없이도 서빙될 수 있어, filesystem 체크는 false negative를 낸다. `.codegraph/` 디렉터리를 `ls`/`test`/`Glob` 등으로 확인하는 경로는 거부하고 오직 `codegraph_status` 응답만 사용한다.
- 응답 텍스트를 아래 "3-상태 분류 기준"으로 분류해 Step 2로 분기한다.

### Step 2 — 분기 (3-상태)

- **(a) healthy** → codegraph 체인 사용:
  1. `codegraph_context <대상>` — 대상 주변 구조적 컨텍스트
  2. `codegraph_trace <from> <to>` — 호출 경로 (필요 시)
  3. `codegraph_impact <대상>` — blast radius
  - 결과를 Step 3의 F6 체크리스트로 정리. 탐색 방법 레이블 = `codegraph (MCP healthy)`.
- **(b) not_initialized** → 사용자에게 **`codegraph init -i` 실행을 제안하는 메시지만 출력**하고, 곧바로 폴백(c)으로 진행한다. **`codegraph init -i`를 자동 실행하지 않는다** (제안에 그친다 — 비목표).
- **(c) unavailable** → 폴백 grep 절차로 진행한다 (Step 3-fallback). 탐색 방법 레이블 = `grep 근사 (codegraph 미사용)`.

> 분기를 트리거하는 MCP 호출 자체는 LLM만 할 수 있고, status 텍스트 → 3-상태 분류는 결정적이다. 분류 기준은 아래 표를 따른다.

### Step 3 — F6 커버리지 체크리스트로 surface

healthy 경로든 grep 폴백 경로든 **동일한 F6 섹션 1~6 + "탐색 방법" 레이블**을 낸다. 양식은 아래 "F6 커버리지 체크리스트" 섹션 참조. (보조: 결정적 코어 `hooks/lib/code_mapper/`의 `check_format_completeness()`로 섹션 누락/순서를 자가검사할 수 있으나, 코어는 비필수 — LLM이 절차만으로 동작 가능.)

### Step 4 — 폐기

산출물을 컨텍스트로 제공한 뒤 **저장하지 않는다.** 파일 쓰기 0, 디스크 산출물 0. 다음 호출에서 재생성된다.

---

## 3-상태 분류 기준 (결정적)

> 이 표가 3-상태 분류의 **SSOT**다. 결정적 코어 `hooks/lib/code_mapper/`의 `classify_probe_state(text)`는 이 기준을 따르며, 코어는 **보조 검증용**이다 (런타임 필수 의존 아님 — LLM이 이 표만으로 분기 가능). 분류는 특정 텍스트 정확매치가 아니라 **의미 단위**로 한다.

| `codegraph_status` 응답 | 상태 | 행동 |
|--------------------------|------|------|
| 인덱스 통계(노드/엣지 수, ready 등)를 반환 | `healthy` | codegraph 체인: context → trace → impact |
| `not initialized`(또는 동등한 미초기화 신호)를 포함 | `not_initialized` | `codegraph init -i` 제안 메시지 출력 + 폴백(c)으로 진행 |
| 도구 미등록(tool not found) / 연결 오류 / 미지 신호 | `unavailable` | grep 폴백 |

> **fail-safe**: 빈 응답·해석 불가·미지 신호는 `unavailable`로 수렴해 grep 폴백을 탄다 (최악도 동작 보장). 잘못 폴백하는 비용 < 잘못 healthy로 판정하는 비용.

## 폴백 grep 패턴 (범용 — 레포 하드코딩 금지)

codegraph 미가용 시(상태 b·c) 아래 순서로 구조적 grep을 실행한다. 패턴은 일반 언어 패턴만 사용하고, 특정 레포 경로·빌드 명령은 일절 쓰지 않는다. `<sym>`은 대상 심볼로 치환한다.

### 1. 언어 판정 (확장자 → 정의 패턴)

대상 파일 확장자로 언어를 판정해 정의 탐색 패턴을 고른다. 미지 확장자는 `generic`으로 폴백한다. Glob으로 정의 파일을 찾는다.

| 확장자 | 언어 | 정의 패턴 (개념) |
|--------|------|------------------|
| `.py` | python | `^\s*(def\|class)\s+<sym>\b` |
| `.ts` `.tsx` `.js` `.jsx` `.mjs` | js/ts | `(function\s+<sym>\b\|class\s+<sym>\b\|(const\|let)\s+<sym>\s*=.*=>\|<sym>\s*\()` |
| 기타 | generic | 식별자 경계 패턴 `\b<sym>\b` (정의/사용 구분 약함 — 근사임을 명시) |

### 2. Callers 근사 (역참조)

대상 심볼을 **import / 호출 / 상속**하는 파일을 grep으로 수집한다:
- import: `import .*<sym>` / `from .* import .*<sym>` / `require\(.*<sym>` / `import \{[^}]*<sym>`
- 호출: `\b<sym>\s*\(` (함수/메서드 호출)
- 상속: `class\s+\w+\(.*<sym>` (py) / `extends\s+<sym>` (ts/js)

### 3. Callees 근사

정의 파일 **본문 범위 내**에서 호출되는 외부 식별자를 grep한다: `\b\w+\s*\(` 매칭 후 정의 심볼 자신을 제외 (best-effort 근사).

### 4. 거짓양성 억제 (정체성 직결 — 가짜 확신 차단)

callers/callees 수집 시 **실제 호출·import·상속·메서드 호출만** 집계하고 아래는 제외한다. 제외 패턴은 코어 테이블이 보유하되, "이 라인이 정말 주석/문자열인가"의 **적용 판단은 LLM이 grep 결과를 보고 수행**한다.

- **주석 제외**: 라인이 주석 마커(`#`, `//`, `/* … */`, `*`)로 시작/감싸이면 제외.
- **문자열 리터럴 제외**: 매치가 따옴표(`"` `'` `` ` ``)로 감싸인 경우 제외.
- **타입힌트 제외**: `: <sym>` / `-> <sym>` 위치 매치는 호출이 아니므로 callers/callees에서 제외 (컨텍스트 참고로만).
- **언어 예약어/제어구문 제외**: callees grep(`\b\w+\s*\(`)이 `if`/`for`/`while`/`switch`/`catch` 등 키워드를 호출로 잡지 않도록 예약어를 제외한다.

### 5. 폴백 출력 규약

- F6 섹션 1~6을 **동일하게** 출력한다.
- "탐색 방법" 레이블 = `grep 근사 (codegraph 미사용)`.
- 각 근사 섹션에 `(grep 근사, codegraph 미사용)` 표기를 붙인다.
- 산출물 말미에 **`근사값 — 개발자 확인 권장`** 주의를 명시한다 (폴백 정확도는 best-effort).
- 근사 결과가 비어도 섹션은 `(없음)`으로 채운다 (생략 금지).

## F6 커버리지 체크리스트 (ephemeral 출력 양식)

매 호출마다 surface해야 할 **커버리지 항목 체크리스트**다 (저장되는 문서 양식 아님). 빠뜨림 — 특히 callers/영향범위 미확인 = 원래 실패 — 방지가 목적이다. **섹션 순서(1~6)와 레이블을 변경하지 않는다.**

```markdown
## 코드맵: <심볼-또는-파일>

### 1. 진입점
- 심볼/파일명:
- 위치(파일:라인):
- 역할 한줄 요약:

### 2. Callers (이 심볼을 호출하는 곳)
| 호출자 | 파일:라인 | 호출 목적 |
|--------|-----------|-----------|
| ...    | ...       | ...       |

### 3. Callees (이 심볼이 호출하는 것)
| 피호출자 | 파일:라인 | 호출 목적 |
|---------|-----------|-----------|
| ...     | ...       | ...       |

### 4. 호출 경로 (Trace)
진입점 → [중간 심볼] → 최종 도달점
(codegraph_trace 결과 또는 grep 근사)

### 5. Blast Radius (영향범위)
이 심볼을 변경하면 영향 받는 파일/심볼 목록:
- ...

### 6. 건드릴 파일 (이번 작업 추정)
이번 작업에서 수정이 필요해 보이는 파일 (추정 — 컨텍스트일 뿐, 계약 아님):
- <파일경로> — 수정 이유

---
탐색 방법: codegraph (MCP healthy) | grep 근사 (codegraph 미사용)
```

### 양식 규약

- **섹션 5 vs 섹션 6 구분**:
  - **섹션 5 (Blast Radius) = 변경 시 영향받는 전체 범위.** codegraph_impact 결과 / 폴백은 callers + 직접 callees 파일.
  - **섹션 6 (건드릴 파일) = 이번 작업에서 수정해 보이는 파일 추정.** 섹션 5의 서브셋이며 컨텍스트일 뿐 **계약이 아니다.**
- **빈 섹션은 `(없음)`** 으로 채우고 생략하지 않는다.
- **순서 변경 금지** — 1~6 순서와 레이블 고정.
- **"탐색 방법" 레이블은 항상 포함** — `codegraph (MCP healthy)` 또는 `grep 근사 (codegraph 미사용)`.
- codegraph 경로와 grep 폴백 경로가 **동일한 섹션 1~6 + 레이블**을 낸다 (포맷 불변량).

## 비목표 (정체성 보호)

- **영속 금지 (ephemeral 전용)**: 코드맵을 파일로 저장하거나 관리 대상 문서로 남기지 않는다. `docs/`·task 문서에 기록하지 않는다.
- **계약 금지**: 산출물을 영속 계약으로 강제하지 않는다 (섹션 6은 추정이지 계약이 아니다).
- **검증 금지**: 구현 diff가 blast radius 안에 머물렀는지 reviewer가 검증하지 않는다.
- **종속 금지**: Phase 3 DAG/Wave의 파일 소유권 입력으로 코드맵에 종속시키지 않는다.
- **하드게이트 금지**: 코드맵 미생성을 이유로 작업을 차단·실패 처리하지 않는다. SDD 에이전트 안내는 권고이지 게이트가 아니다.
- **시각화 도구 구현 금지**: 그래프 렌더링·다이어그램 생성·UI 출력은 범위 밖.
- **AST 엔진 신규 구현 금지**: codegraph MCP와 grep/Glob을 오케스트레이션할 뿐, 자체 AST 파서·심볼 인덱스·정적 분석 엔진을 구현하지 않는다.
- **codegraph 자동 init 금지**: `codegraph init -i`는 제안(안내 메시지)에 그치며 자동 실행하지 않는다.
- **전이적 의존 완전 추적 금지**: blast radius는 직접 호출 관계 1~2단계로 제한한다 (전체 재귀 탐색 아님).
- **레포 특화 하드코딩 금지**: 특정 레포 빌드 명령·경로·컨벤션을 이 절차에 기술하지 않는다.
- **protected set 자동 수정 금지**: `self-improve`, `pr-converge`, `harness-improvement-critic`, 게이트 스크립트를 자동 생성/수정하지 않는다.

> 이들은 업계가 스테일·과잉제약 비용으로 회피한 패턴이다. 필요 시 별도 측정 후 v2 실험으로만 검토한다.

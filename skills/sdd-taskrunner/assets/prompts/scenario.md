# 테스트 시나리오 생성 프롬프트

다음 태스크에 대한 TDD 테스트 시나리오를 작성하세요.

## 입력
- 태스크: {{task_title}}
- 완료 조건: {{completion_conditions}}
- 데이터 모델/인터페이스: {{interfaces}}
- 관련 spec 요구사항: {{spec_requirements}}

## 시나리오 작성 원칙

1. **공개 API만 테스트** — 내부 구현 방식을 가정하지 않는다
2. **3가지 카테고리 필수**:
   - Happy path (정상 동작) — 최소 2개
   - Edge case (경계값) — 최소 1개
   - Error case (에러 처리) — 최소 1개
3. **입력과 기대 결과를 구체적으로** — "올바른 값" 대신 실제 예시
4. **유형 명시** — unit (단위), integration (통합)

## 출력 형식

| # | 시나리오 | 입력 | 기대 결과 | 유형 |
|---|----------|------|----------|------|
| 1 | 한국어 텍스트 토큰 추정 | "안녕하세요" (5글자) | 8 토큰 (5 × 1.5 = 7.5 → ceil 8) | unit |
| 2 | 빈 문자열 처리 | "" | 0 토큰 | unit |
| 3 | 혼합 텍스트 (한국어+영문) | "Hello 세계" | 한국어 2글자(3) + 영문 1단어(2) = 5 토큰 | unit |
| 4 | 존재하지 않는 모델 ID | "unknown-model" | 기본 한도 200000 반환 | unit |

# Adversarial Review

기존 리뷰 이터레이션으로 해결되지 않는 문제를 **근본적으로 재검토**하는 스킬.
체크리스트 수준의 리뷰가 아니라, 접근법 자체를 의심하고 대안을 제시한다.

## 사용 시점

- 코드 리뷰 피드백이 반복되는데 해결이 안 될 때
- 구현 방향 자체가 맞는지 의심될 때
- 외부에서 직접 호출: `/adversarial-review`

## 프로세스

1. **호출자가 제공하는 입력**:
   - 변경 파일 경로 목록 + diff
   - 기존 리뷰 피드백 이력 (있으면)
   - 태스크/요구사항 문서 (있으면)

2. **adversarial-reviewer 에이전트 디스패치** (opus):
   - 근본 원인 진단
   - 접근법 비판 + 대안 제시
   - 비판 수준 판정: 근본적 / 구조적 / 니트픽

3. **이터레이션 루프**:
   - REWORK → 대안을 구현자에게 주입 → 구현자 수정 → 재비판
   - PASS → 지적이 니트픽 수준으로 떨어짐 → 루프 탈출
   - BLOCKED → 대안 제시 불가 → 호출자에게 보고

## 비판 수준

| 수준 | 의미 | 행동 |
|------|------|------|
| **근본적** | 접근법/아키텍처가 잘못됨 | 대안 방향 제시 → 구현자 재작성 |
| **구조적** | 방향은 맞지만 구현 구조에 문제 | 구체적 리팩터링 지시 |
| **니트픽** | 사소한 개선 여지 | **PASS** — 충분히 좋음 |
